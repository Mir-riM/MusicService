# Music Service

Пет-проект с музыкальным сервисом.

Изначально проект не планировался для публикации на GitHub, поэтому история коммитов начинается не с самого начала.

Проект находится в разработке.

---

## Запуск

```

# Через docker
docker compose up --build

# Вручную (не советую, так как нужен minio и mongodb)
npm run dev

```

## Описание

Веб-сервис для работы с музыкой.
На текущий момент реализованы следующие возможности:

- прослушивание треков
- загрузка музыки
- поиск треков

---

## Технологии

### Frontend

- Next.js
- Material UI
- Tailwind CSS
- Redux Toolkit
- RTK Query

### Backend

- NestJS
- Passport
- Guards
- JWT

### Хранилище и БД

- MongoDB
- MinIO (S3)

Для MinIO необходимо создать публичные бакеты:

- `tracks` — аудиофайлы
- `pictures` — изображения

---

## Инфраструктура

Для всех сервисов используется **docker-compose**.

В `docker-compose` описаны и связаны между собой:

- frontend (Next.js)
- backend (NestJS)
- MongoDB
- MinIO (S3)

---

## Авторизация

Используется авторизация на основе:

- access token
- refresh token

---

## Планы

В дальнейшем планируется:

- плейлисты (создание, отслеживание, форк)
- добавление понравившихся треков
- удаление треков, загруженных пользователем
- модерация (admin / moderator)
- переключение треков в плеере
- случайное воспроизведение
